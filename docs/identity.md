# clickhousex 身份

## 我是谁

`clickhousex` 是 FoundationX 的 **ClickHouse 存储扩展模块**，提供 ClickHouse 列式数据库客户端封装。

## 我做什么

- ClickHouse 连接池和健康检查
- 批量写入和查询接口封装
- 配置模型标准化

## 我不做什么

- 不是分析业务逻辑 — 分析语义由调用方定义
- 不是模板源 — 模板生成属于 xlib-standard
- 不依赖其他存储模块

## 与 taosx 的时序存储边界

`clickhousex` 与 `taosx` 都处理时序数据，但面向不同场景，互补不重叠：

| 维度 | clickhousex | taosx |
|------|-------------|-------|
| 存储引擎 | ClickHouse | TDengine |
| 面向场景 | **OLAP 分析查询** | **IoT 时序存储** |
| 典型负载 | 聚合查询、多维分析、即席查询 | 高频写入、时间窗口查询、设备/行情数据 |
| 数据特点 | 宽表、星型模型、历史归档 | 流式写入、超级表、按时间降采样 |
| 选型依据 | 需要灵活聚合和快速即席分析的场景 | 需要高吞吐写入和简单窗口查询的场景 |

两者同属基座存储扩展层，功能互补，不在同一维度竞争。
